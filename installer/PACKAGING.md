# SnipEasy packaging notes

## Prerequisites

- [.NET 9 SDK](https://dotnet.microsoft.com/download/dotnet/9.0)
- [Inno Setup 6](https://jrsoftware.org/isinfo.php) — free, used to build the graphical installer wizard. Install to the default location or add `iscc.exe` to PATH.
- `installer/ChineseSimplified.isl` — Chinese Simplified translation file for Inno Setup, bundled in the repo since it is not included in the default Inno Setup installation.

## Current supported package

- `installer/Build-SnipEasy-Setup.ps1` builds the Inno Setup installer used by the website download. It stages the Release output, finds `iscc.exe`, and compiles `installer/SnipEasy.iss` into `website/downloads/SnipEasy-Setup.exe`.
- `installer/SnipEasy.iss` is the Inno Setup script defining the installer wizard (welcome, license, path selection, progress, finish page) and uninstall support.
- `installer/Publish-SnipEasy.ps1` creates repeatable framework-dependent or self-contained publish output and an update manifest.

## Building the installer

```powershell
# 1. Build in Release mode
dotnet build SnipEasy.sln -c Release

# 2. Build installer (requires Inno Setup 6)
powershell -NoProfile -ExecutionPolicy Bypass -File installer/Build-SnipEasy-Setup.ps1

# Without ffmpeg (degraded, no MP4/audio recording):
powershell -NoProfile -ExecutionPolicy Bypass -File installer/Build-SnipEasy-Setup.ps1 -AllowNoFfmpeg
```

The output is `website/downloads/SnipEasy-Setup.exe`.

## Signing

Run from a Windows SDK Developer Command Prompt or provide `signtool.exe` on PATH:

```powershell
powershell -NoProfile -ExecutionPolicy Bypass -File installer/Publish-SnipEasy.ps1 -SelfContained -CertificatePath path\to\certificate.pfx -CertificatePassword "..."

# Sign the final installer. Prefer setting SNIPEASY_CERT_PASSWORD in the CI secret store.
powershell -NoProfile -ExecutionPolicy Bypass -File installer/Build-SnipEasy-Setup.ps1 -CertificatePath path\to\certificate.pfx
```

Never commit `.pfx`, `.p12`, or other private signing credentials to the repository.

## MSIX/MSI path

The app is ready for MSIX/MSI packaging once a signing certificate and identity are chosen. Keep `%LOCALAPPDATA%/SnipEasy` as the user-data location and preserve it across upgrades/uninstalls unless the user explicitly chooses to remove user data.

## Update manifest

`update-manifest.json` is generated by `Publish-SnipEasy.ps1` and can be hosted next to the installer. The app includes a local manifest parser so enterprise deployments can point clients at a managed manifest file in a future UI setting.
